Student Management System

A Student Management System (SMS) rule is a guideline for the system's operation, ensuring data integrity, security, and functionality. Key rules include creating a well-structured, verifiable Software Requirements Specification (SRS) document that focuses on the system's external behavior, handling user roles, securing data through OAuth2 or LDAP integration, and managing student records and processes like admissions, attendance, and grading
